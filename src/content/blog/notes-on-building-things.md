# Notes on building things

I like products that feel *coherent*.

That usually means:

1. One visual language
2. One mental model
3. Small interactions that make the app feel intentional

## Why the Finder metaphor is useful here

The blog is not just a list of links. It is a small file browser:

- you can sort posts
- scan summaries quickly
- open a note and read it without leaving the app shell

## A tiny rule of thumb

If the UI suggests a metaphor, it should commit to it.

For a Finder-like surface, that means:

- folders
- files
- sorting
- previews

And from the terminal side, `ls` is the natural bridge back into the same content.
